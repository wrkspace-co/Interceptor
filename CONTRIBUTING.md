# Contributing

Thanks for your interest in contributing to Interceptor.

## Getting Started
- Fork the repo and create a branch from `main`.
- Install dependencies: `pnpm install`.
- Run tests: `pnpm test`.
- Run a local build: `pnpm build`.

## Development Workflow
1. Create a feature branch: `git checkout -b feature/your-feature`.
2. Make your changes with focused commits.
3. Ensure tests pass and add tests for new behavior.
4. Open a pull request with a clear description.

## Project Conventions
- TypeScript-first. Keep types accurate and minimal.
- Prefer small, focused modules over monolithic files.
- Avoid breaking changes unless explicitly discussed.
- Keep public APIs documented in `README.md`.

## Testing
- Unit tests are in `tests/`.
- Run the full suite with `pnpm test`.

## Reporting Issues
- Use GitHub Issues for bugs and feature requests.
- Include steps to reproduce and environment details.

## Code of Conduct
By participating, you agree to abide by the Code of Conduct.
